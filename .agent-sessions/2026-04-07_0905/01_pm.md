## 3. Top 3 Picks

### #1 — Fix the Poll Route (ROI: 4.00)
**A one-line router change that immediately makes the vibe assessment quiz reachable.**
The poll is the *lightest* psychoanalytic entry point — 4 questions, 5 archetype outputs, no auth required. It's the perfect top-of-funnel hook and it's currently silently redirecting users to the homepage. This is money left on the table.
- **Scope: S** — Router fix + add nav entry point from HomeView

### #2 — Onboarding Flow (ROI: 2.50)
**A guided post-login sequence that walks new users from `/calibrate` through `/psychoanalysis` to `/intake` in a single coherent arc.**
Right now a new user authenticates and has no idea what to do. The pipeline exists (`calibrate → peripheral → psychoanalysis → intake → game`) but is invisible. Without this, the matching engine is effectively unreachable for anyone who didn't build it.
- **Scope: M** — Progress tracker component, step validation, skip logic, first-run detection flag

### #3 — Register Fitting.vue in Router (ROI: 3.00)
**Add `/fitting` to the router and surface it from the matching flow or profile.**
The 3D swimsuit fitting room is a fully built, visually distinctive differentiator. It's completely orphaned. Even as a standalone "playground" link it adds perceived depth to the product and signals that ChannelZero is not just another quiz app.
- **Scope: S** — Router entry + nav link + optional auth guard decision

## 5. Risk Flags

**Blocker for onboarding (#2):**
- `useAuthStore` drives the auth guard but there's no `isFirstRun` or `onboardingComplete` flag in the schema. Adding this requires a backend migration (users table) + store update. Not hard, but it's not a zero-effort change.

**Blocker for match action flow (future top pick):**
- `GameView` renders a match profile but there's no messaging, accept/decline, or queue system in the backend. This is a full-feature build, not a UI fix. Don't touch this until onboarding is working — you need users in the funnel first.

**Letterboxd dependency:**
- `PeripheralSync` silently 503s on Letterboxd calls. Users who try to connect it will see a failure with no explanation. Add a disabled state + "coming soon" label to that tile until the API key arrives.

**Orphaned component debt:**
- `Confessions.vue` and `OracleIntake.vue` at the project root are stale shadows of `IntakeView` and `GameView`. If these are not intentionally in progress, delete them — they create confusion about which component is canonical.

**No `/poll` recovery:**
- PollView exists, works, and is psychologically the strongest first impression in the app. Every day the route redirects to `/` is a lost signal-collection opportunity.
